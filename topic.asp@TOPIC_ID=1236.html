А Вам обязательно нужно при нажатии? Двойной клик не подойдет?
Обычно такие вещи делаются по двойному клику. Для этого в Format / Bahavior на вкладке Double-Click ставится галочка в Go to page и выбирается нужная страница.
А по одиночному клику Вам придется либо отлавливать событие SelectionChanged, либо работать с мышиными событиями.
Здесь механизм будет сложнее.
Можно учесть, что при нажатии на шейп он селектируется и обрабатывать событие SelectionChanged.
Пишем обработчик
Private Sub Win_SelectionChanged(ByVal Window As IVWindow)
If Window.Selection.Count > 0 Then
...
End If
End Sub
Вместо многоточия нужно подставить код, который прочитает из какой-то ячейки (например из Custom Properties шейпа) имя страницы для перехода и выполнит собственно переход. А предварительно еще и проверит, относится ли селектированный шейп к тем, с которого следует осуществляеть переход.
Но этот обработчик сразу не заработает. Предварительно нужно объявить окно WithEvents. А при открыти документа или отдельным макросом связать объявленный win с активным окном, типа вот так:
Dim WithEvents Win As Window
Sub ttt()
Set Win = ActiveWindow
End Sub
Если сначала выполнить макрос ttt, то после этого начнет срабатывать Win_SelectionChanged и щелчок по шейпу будет вызывать переход на другую страницу.
